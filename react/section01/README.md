## 2. 변수와 상수

### 변수

- 계속 바뀌는 값을 저장하는 이름 붙은 저장소
- 변수명 규칙
  - 변수명에 기호 사용 불가능 (’\_’ , ‘$’ 는 사용가능)
  - 변수명은 숫자가 아닌 문자로 시작해야 함
  - 예약어 피하기 (if와 같은 변수명 금지)
- let, var로 선언 가능
  - var로 선언한 변수는 여러번 선언 가능, let은 불가능

### 상수

- 바뀌지 않는 값을 저장하는 이름 붙은 저장소
- const로 선언 가능
- 읽기만 가능 (선언 이후에 값 변경 불가 ⇒ 선언과 동시에 초기화 안 할 시 에러)

## 3. 자료형과 형 변환

### 자료형

- Primitive Type (원시 타입)
  - 한 번에 하나의 값만 가질 수 있음
  - 하나의 고정된 저장 공간 이용
  ```jsx
  let number = 12;
  ```
  - 숫자형 타입
    - 정수형, 실수형, Infinity, -Infinity, NaN
  - 문자열 타입
    - “”, ‘’, ``로 감싸서 표현
    - 템플릿 리터럴 : `${변수명}`
  - 불리언 타입
    - true, false
  - null
    - 의도적으로 대입해야됨
  - undefined
    - 초기화 안 하면 기본값
- Non-Primitive Type (비 원시 타입)
  - 한 번에 여러 개의 값을 가질 수 있음
  - 여러 개의 고정되지 않은 동적 공간 사용
  ```jsx
  let array = [1, 2, 3, 4];
  ```

### 형 변환

- 값을 유지하면서 자료형을 변환시키는 기능
- 묵시적 형변환
  - 숫자 \* 문자 ⇒ 문자를 숫자로 바꿈
  - 숫자 + 문자 ⇒ 숫자를 문자로 바꿈
- 명시적 형변환
  - parseInt

## 7. 함수 표현식 & 화살표 함수

- 자바스크립트에선 함수도 하나의 값으로 취급하여 변수나 상수에 담아서 사용 가능

```jsx
// 함수 표현식
let 변수명 = function () {
  내용;
};

// 화살표 함수
let 변수명 = () => {
  내용;
};

// return 줄 한 줄만 있으면 중괄호, return 생략가능
let 변수명 = () => 내용;
```

- 함수 선언식으로 선언한 함수는 호이스팅이 이뤄져서 선언식보다 위에서도 사용가능
- 함수 표현식, 화살표 함수는 호이스팅이 이뤄지지 않아 변수가 초기화된 이후에 사용가능

## 9. 객체

```jsx
// 객체 생성자 방식
let person = new Object();

// 객체 리터럴 방식
let person = {};

let person = {
	key: "value" // 프로퍼티 (객체 프로퍼티)
	// value는 모든 자료형 가능
	// key는 중복 가능/ 문자열만 가능
};

let person = {
	name: "황주영",
	age: 29,
}

// 점 표기법
person.name;
// 괄호 표기법
person["age"];

// 프로퍼티 추가, 수정
person.location = "한국";
person["gender"] = "남성";

// 프로퍼티 삭제
delete person["name"];
// 이 방법은 프로퍼티와 객체의 연결은 끊지만 메모리는 남아있음
person["name"] = null;
// 삭제한 것과 똑같은 효과 + 문자열을 메모리에서 날릴 수 있음

// 프로퍼티로 함수도 가능 => 메소드라고 부름 / 함수가 아닌 프로퍼티 => 멤버라고 부름
const person = {
	say = function () {
		name: "황주영",
		age: 29,
		console.log(`안녕 나는 ${this.name}`); // 안녕 나는 황주영
		// this는 객체 자기 자신
	}
};

// in 연산자를 통하여 객체 안에 해당 프로퍼티가 있는지 확인 가능
console.log("name" in person); // true

```

- 객체는 상수로 선언하여도 프로퍼티 추가, 수정 가능
  - 객체 자체를 바꾸는 것이 아니기 때문
  - 객체 자체를 재할당하는 것은 불가능
